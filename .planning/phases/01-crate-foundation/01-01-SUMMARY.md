## Decisions Made

- Directory-form module (src/deploy/mod.rs) used instead of flat src/deploy.rs so Phase 2 can add primary.rs and fallback.rs as siblings without restructuring
- clap features = ["derive"] listed in Phase 1 even though no clap code exists — avoids a confusing Cargo.toml-only edit in Phase 2
- NonZeroExit uses named struct variant `{ stderr: String }` rather than tuple form — stderr field name is self-documenting
